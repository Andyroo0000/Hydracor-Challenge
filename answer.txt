d) I mentioned this a little but changes from using a list of dictionaries to hold the media item data to using classes was big for me. Using classes and objects creates a much more organized and scalable program. Unlike dictionaries where it's hard to scale and read it sometimes. Creating functions within my class makes the proability I mispell or something along those lines less likey as you can do object.name. Like I said scalability, I can create multible different media types and if I wanted to make a little change I can just inherit from the parent and add a function or change a attribute. If I used dictionaries I would have to make a whole new one and if your doing this at a large scale it's hard to manage and do.

e) There are trade offs though. One such was if I just used dictionaries when transfering the data to a json file, I wouldn't need to unload the object data and turn it into a dictionary. This process could have been a little easier for me. Also the convience, using classes give great structure but if I just wanted to quickly add something to a dictionary I can just add it whilst using classes I would need a constructor or change methods and functions within the class. Overall, depsite these downsides classes make programing this much better than it would be using dictionaries.
